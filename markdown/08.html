8. 一定の学習後に性能を上げたい場合

この章では、Google DriveとGoogle ColaboとCommand Promptを用いて作業します

動画全体での推論時に誤判別につながっていると考えられるフレームを抜き出して、再びラベル付けをして、追加学習させる、ということをしてみます。

8.1 Google Driveからプロジェクトフォルダをダウンロード

これまでに取り組んでいたプロジェクトフォルダをダウンロードして、手元のプロジェクトと置き換えます。もともと手元にあったプロジェクトは削除して構いません。

8.2 管理者としてCommand Promptを起動、仮想環境を立ち上げる

1. 仮想環境の立ち上げと、* 2. プロジェクトの作成の2.2までと同じことをします。

8.3 誤判別につながっていると考えられるフレームを抽出

New: まず、deeplabcut.extract_outlier_frames()関数で、ビデオ推論時において誤判別につながっていると考えられるフレームを抽出します。どのフレームが誤判別につながっているかを自動で判断する方法は大きく分けて3つあります。1つは、推論時の予測尤度が低いフレームを誤判別とする方法、2つ目は、ある体の部位の座標が移動平均による推定値よりも大きく外れているフレームを誤判別とする方法、3つ目は、ある体の部位の座標が直前の位置よりも大きく変化（ジャンプ）しているフレームを誤判別とする方法です。

これらの3つの方法は、deeplabcut.extract_outlier_frames()関数のoutlieralgorithmという引数を変更することで切り替えることができます。

  • 推論時の予測尤度が低いものを抽出したい時にはoutlieralgorithm = 'uncertain'
    • 尤度の基準は、p_boundという引数で指定します。デフォルトはp_bound = 0.01です。
  • ある体の部位の座標が移動平均による推定値よりも大きく外れているフレームを抽出したい時にはoutlieralgorithm = 'Fitting'
    • どれくらい外れているかの基準はepsilonという引数で指定します。デフォルトはepsilon = 20です。
  • ある体の部位の座標が直前の位置よりも大きく変化しているものを抽出したい時にはoutlieralgorithm = 'jump'
    • どれくらいジャンプしたかの基準はepsilonという引数で指定します。デフォルトはepsilon = 20です。

としてください。もちろん、outlieralgorithm = 'manual'で手動で選ぶこともできます。ちなみに、デフォルトはoutlieralgorithm = 'jump'です。デフォルトの設定で十分である場合が多いと思います。詳しくはこちらを参考にしてください。

import os

# 変更対象
path_project = os.getcwd() + "\\task-researcher-yyyy-mm-dd" # それぞれ変更してください
video_name_list = [ '*****.avi', '*****.avi'] # *****をビデオファイル名にしてください

# Don't edit
path_config = path_project + '\\config.yaml'
video_path_list = list(map(lambda x: path_project+"\\videos\\"+x, video_name_list))

# 変更可能
deeplabcut.extract_outlier_frames(path_config, video_path_list, outlieralgorithm = 'jump') # 'jump'がでフォルト

学習を改善したいビデオについて、yesと入力して進んでいってください。実行に時間が少々かかります。

8.4 ラベルの位置を変える

deeplabcut.refine_labels()関数で、誤判別につながっていると考えられるフレームについて、ラベルの位置を調整します。

deeplabcut.refine_labels(path_config)

GUIが立ち上がります。今回はラベル付けとは異なり、フォルダではなく、h5ファイルを選択します。

まず、改善したい動画と同じ名前のフォルダをクリックします。そこに、machinelabels-iter0.h5というファイルがあります。これを選択して、右下のOpenを押してください。（参考 : 1回目のColabでのtrainに関するものはiter0、n回目のColabでのtrainに関するものはiter{n-1}です）

すると、尤度の閾値を聞かれます。詳細のことは確認できていませんが、とりあえずデフォルトの0.40のままでいいと思います。

次に、学習によって予測された位置が自分の意図する位置と一致するかを確認します。Nextでフレームを切り替えて、最後までやってください。

8.5 修正したデータを教師データとして付け加える

deeplabcut.merge_datasets()関数で、修正したデータを教師データとして付け加えます。

deeplabcut.merge_datasets(path_config)

こうして、追加で学習ができるようになりました。

8.6 新しい動画を教師データとして追加したい場合

新しい動画を教師データとして追加したい場合には、deeplabcut.add_new_videos()関数を用います。

video_path_list_add = [ 'C:\\*****\\*****.avi' ] # フルパス

deeplabcut.add_new_videos(path_config, video_path_list_add, copy_videos = False)

あとは、4. フレームの抽出とラベル付けからと同じように、新たに追加した動画について、フレームの抽出とラベル付けを行います。

8.7 Google Driveにアップロードする

作業したプロジェクトフォルダ task-researcher-yyyy-mm-dd をGoogle Driveにアップロードします。

これ以降はまた7. Google Colaboratoryと同様の作業をします。
